Once the RDF files have been generated (see the [cisreg](../cisreg/) section), the resulting triples can be loaded into a triple store such as Virtuoso or GrapDB. Then, the network can be explored using the [SPARQL](https://www.w3.org/TR/sparql11-query/) query language. A dedicated [tutorial](../docs/SPARQL_Tutorial.pdf) is provided in the repository to help users become familiar with this language and the network.

BioGateway was hosted on a [Virtuoso](https://github.com/openlink/virtuoso-opensource) instance, and the endpoint [https://semantics.inf.um.es/biogateway](https://semantics.inf.um.es/biogateway) is available for use. The use cases hosted in this directory were run on that endpoint.

In addition, the interactive web application [INTUITION](https://semantics.inf.um.es/intuition_biogateway/) uses this endpoint as source to enable users to generate interactive queries without prior skills in SPARQL. A [tutorial](../docs/INTUITION_Tutorial.pdf) on how to use it is also available. In addition, the endpoint can also be accessed programmatically via R and Python libraries (beta versions): [https://github.com/tecnomod-um/RBioGateway](https://github.com/tecnomod-um/RBioGateway) and [https://github.com/tecnomod-um/PyBioGateway](https://github.com/tecnomod-um/PyBioGateway). These libraries also generate queries automatically, so they eliminate the need to create SPARQL queries manually.
